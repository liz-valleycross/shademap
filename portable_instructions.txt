=====================================

This tool MUST run on localhost (local web server).

EASIEST OPTION - PORTABLE SERVER (No Install Required!)
========================================================

Download one of these portable servers (pick ONE):

OPTION 1: Caddy (RECOMMENDED - Easiest)
----------------------------------------
‚úì Single executable, no installation
‚úì Works immediately

   - Select: Platform = Windows, Package = GitHub

HOW TO USE THE APP:
===================
1. Start the server (double-click start_server.bat)
2. Browser opens automatically
3. Search for your address
4. Switch to "Satellite" view (easier to see parking lots)
5. Enter trailer dimensions:
   - Width (feet)
   - Length (feet)
6. Click "Draw Parking Lot Area"
   - Click corners of the parking lot
   - Double-click to finish
7. Enter your Shadow Map API key
   - Get free key from: https://shademap.app/about/
8. Set analysis dates (start and end)
9. Click "Analyze Parking Spots"
10. View results ranked by sun exposure!

Results show:
- Ranking (#1 = best spot)
- Sun exposure percentage
- Estimated hours per day of direct sun
- Exact coordinates
- Red boxes on map showing trailer placement

TROUBLESHOOTING:
================

"Server not found" error:
‚Üí Download caddy.exe or install Python

Port 8000 already in use:
‚Üí Close other programs using port 8000
‚Üí Or modify start_server.bat to use port 8080

Can't find address:
‚Üí Use full address including zip code
‚Üí Example: "123 Main St, Orlando, FL 32801"

Map not loading:
‚Üí Check internet connection
‚Üí Libraries load from CDN, need internet

Analysis not working:
‚Üí Verify Shadow Map API key is correct
‚Üí Make sure parking lot is drawn
‚Üí Check that dates are valid

Libraries not loading:
‚Üí Check internet connection
‚Üí Try different browser (Chrome recommended)

WHY LOCALHOST IS REQUIRED:
==========================
The app uses:
- External JavaScript libraries (Leaflet, etc.)
- Advanced browser APIs
- Cross-origin requests

These require proper HTTP server (localhost) and won't work
when opened directly as file:// in browser.
